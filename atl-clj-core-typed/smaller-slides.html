core.typed is a type annotation system for clojure.

Tell clojure about types

core.typed is heavily influenced by Typed Racket, a variation of Racket (PLT Scheme).

Type checking is still optional and separate from compilation.

Annotation is better!

The core.typed position is that annotating clojure code with type information provides:

  • Better error messages
  • Better documentation

Union Type

A function that takes either a String or a Named and returns a String.

Option Type

namespace accepts an argument of type String, Symbol, or Keyword and returns type String or nil. Same as

(U java.lang.String nil)

Intersection Type

supers accepts an argument of type Class and returns a either a non empty Set of type Class or type nil.

How does annotation work?

Use the clojure.core.typed/ann macro to define the types used by a function.

[Type -> Type] and (Fn [Type -> Type]) are equivalent.

Filters

core.typed is more than just types. It also collects a filter set for each expression: a true filter (then) and a false filter (else).

Filters are not usually written in normal code.

Filters

  • tt is the trivially true filter
  • ff is the impossible filter

Filters

Filters relate bindings to types.

In the form is Symbol 0, the 0 is the first argument to the function. If the expression is true, the first argument has type Symbol.
